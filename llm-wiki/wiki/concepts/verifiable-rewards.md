# Verifiable Rewards

The source argues that some technical domains improve faster because they provide explicit, checkable reward signals that work well with reinforcement learning [Karpathy x: AI Capability Perception Gap](../sources/karpathy-x.md). The example given is programming tasks that can be judged by objective criteria such as whether unit tests pass. `(high confidence)`

## Contrast In The Source

| Domain type | Reward clarity |
|------|----------------|
| Programming and similar technical tasks | High: pass/fail or otherwise machine-checkable |
| Writing and softer advisory tasks | Lower: quality is harder to score explicitly |

## Why It Matters

This concept is used to explain why capability gains are currently uneven rather than uniform. In the source's framing, clearer reward signals accelerate improvement in technical agentic tasks and contribute to [[peaky-capabilities]].
